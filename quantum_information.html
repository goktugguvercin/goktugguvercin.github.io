Introduction to Quantum Information

Descriptions

Dirac notation is a standard mathematical way used to describe vectors and matrices. It is not exclusive to quantum information; it applies equally well to both classical and quantum information.

There are two mathematical frameworks for describing quantum systems:

State-vector formalism

Quantum states are represented by vectors; operations are represented by unitary matrices.

Density-matrix formalism

Quantum states are represented by density matrices. This framework allows for a more general class of quantum operations and measurements.

The state-vector formalism is simpler and sufficient for understanding most quantum algorithms. However, it is not adequate for describing noisy quantum systems. To model noise, decoherence, and statistical mixtures of states, the density-matrix formalism is required.

Key relationship. The state-vector formalism can be viewed as a special case of the density-matrix formalism.

Classical Information

Imagine a physical system or device that stores information; let's call it as \(X\). We assume that there is a finite set of classical states, and at each moment \(X\) occupies exactly one of them. Here a classical state means a configuration of the system that can be recognised and described unambiguously, without any uncertainty or error.

The Classical State Set

We denote the classical state set by the symbol \(\Sigma\). Two standard examples:

System \(X\) Classical state set \(\Sigma\)
A bit \(\Sigma = \{0, 1\}\)
A 6-sided die \(\Sigma = \{1, 2, 3, 4, 5, 6\}\)
Why \(\Sigma\) must be non-empty. Every information-storing system needs some state to be in. Additionally, to be useful, it needs at least two distinct states, so it can distinguish one possibility from another.

Probabilistic States

Often we do not know the exact classical state of \(X\) with certainty. Suppose that I flip a coin and immediately look at it. The coin is heads, and there is no uncertainty. In the second case, I can flip it again, but I am covering it before I see it. The coin is still either heads or tails underneath. Nevertheless, from our perspective:

\[ P(X = H) = 0.5 \qquad P(X = T) = 0.5 \]
This is a probabilistic state. The physical coin is still in a classical state (heads or tails). The probability distribution represents your lack of knowledge. So "probabilistic state" does not mean the coin is not in a state; it means you do not know which classical state it is in.

Probability Vectors

A probabilistic state is conveniently written as a column vector whose entries are the probabilities, listed in a fixed order of the classical states. The coin example above becomes:

\[ \begin{bmatrix} 0.5 \\ 0.5 \end{bmatrix} \]

The entries are non-negative real numbers that sum to \(1\). This vector is called a probability vector. This is exactly the representation generalised by quantum state vectors.
